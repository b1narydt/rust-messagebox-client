  <files>src/client.rs, src/http_ops.rs</files>
  <behavior>
    - init() via init_once.get_or_try_init: queries existing advertisements, calls anoint_host only if no matching ad exists for current host
    - init_once ensures anoint_host is called at most once even under concurrent access
    - list_messages with no host_override queries all advertised hosts concurrently, deduplicates by message_id
    - list_messages with host_override queries only that host (no dedup needed)
    - Deduplication uses HashMap keyed by message_id — first occurrence wins
    - If all hosts fail, return error; if some fail, return results from successful hosts
  </behavior>
  <action>
1. **Update `assert_initialized` in `src/client.rs`:**
   Replace the current stub implementation with full init-once logic:
   ```
   pub(crate) async fn assert_initialized(&self) -> Result<(), MessageBoxError> {
       // OnceCell ensures this runs exactly once
       self.init_once.get_or_try_init(|| async {
           let identity_key = self.get_identity_key().await?;
           let ads = self.query_advertisements(Some(&identity_key), Some(&self.host)).await?;
           if ads.iter().all(|ad| ad.host.trim() != self.host.trim()) {
               self.anoint_host(&self.host).await?;
           }
           Ok::<(), MessageBoxError>(())
       }).await?;
       Ok(())
   }
   ```
   Remove the old `self.get_identity_key().await?; Ok(())` stub.

2. **Add `pub async fn init(&self) -> Result<(), MessageBoxError>` in `src/client.rs`:**
   Public wrapper that calls `self.assert_initialized().await`. This is the user-facing API.

3. **Update `list_messages` in `src/http_ops.rs` for multi-host dedup:**
   - Add `use std::collections::HashMap;` and `use futures_util::future::join_all;`
   - Before the existing `list_messages` logic, add a multi-host branch:
     - If no host override is being used (the standard path), after `assert_initialized`:
       a. `let identity_key = self.get_identity_key().await?;`
       b. `let ads = self.query_advertisements(Some(&identity_key), None).await.unwrap_or_default();`
       c. Build host set: `HashSet<String>` from ads + self.host
       d. If only 1 host (self.host), use existing single-host path (no change)
       e. If multiple hosts, call a new private helper `list_messages_from_host(host, message_box, accept_payments)` for each host via `join_all`, collect results, deduplicate by `message_id` using `HashMap::entry().or_insert()`
   - Extract the existing single-host list_messages body into `async fn list_messages_from_host(&self, host: &str, message_box: &str, accept_payments: bool) -> Result<Vec<PeerMessage>, MessageBoxError>` — this takes an explicit host URL, builds the request against that host, does payment internalization if `accept_payments` is true.
   - The existing `list_messages` becomes the orchestrator: single host = direct call, multi host = join_all + dedup.
   - TS uses `Promise.allSettled` — Rust equivalent: `join_all` returning `Vec<Result<...>>`, filter with `.into_iter().filter_map(|r| r.ok())`.

4. **Unit tests:**
   - In `http_ops.rs` tests: `test_list_messages_dedup_by_id` — create two `PeerMessage` vecs with overlapping message_ids, merge them using the dedup logic (extract dedup into a testable function `dedup_messages(results: Vec<Vec<PeerMessage>>) -> Vec<PeerMessage>`), verify output has no duplicates.
   - In `client.rs` tests: `test_init_idempotent` — verify `init_once` is of type `OnceCell<()>` and `get_or_try_init` pattern compiles. (Full behavioral test requires overlay which is integration-level.)
