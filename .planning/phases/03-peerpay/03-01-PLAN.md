Add three more methods to the `impl<W>` block in `src/peer_pay.rs`:

1. `pub async fn accept_payment(&self, payment: &IncomingPayment) -> Result<(), MessageBoxError>`:
   - Call `self.wallet().internalize_action(InternalizeActionArgs { tx: payment.token.transaction.clone(), description: "PeerPay Payment".to_string(), labels: vec!["peerpay".to_string()], seek_permission: BooleanDefaultTrue(Some(true)), outputs: vec![InternalizeOutput::WalletPayment { output_index: payment.token.output_index.unwrap_or(0), payment: Payment { derivation_prefix: payment.token.custom_instructions.derivation_prefix.as_bytes().to_vec(), derivation_suffix: payment.token.custom_instructions.derivation_suffix.as_bytes().to_vec(), sender_identity_key: PublicKey::from_string(&payment.sender).map_err(|e| MessageBoxError::Wallet(format!("invalid sender key: {e}")))? } }] }, self.originator()).await.map_err(|e| MessageBoxError::Wallet(e.to_string()))?;`
   - Call `self.acknowledge_message(vec![payment.message_id.clone()]).await?;`

2. `pub async fn reject_payment(&self, payment: &IncomingPayment) -> Result<(), MessageBoxError>`:
   - If `payment.token.amount < 2000`: just call `self.acknowledge_message(vec![payment.message_id.clone()]).await` and return
   - Otherwise: call `self.accept_payment(payment).await?` (this internalizes + acks)
   - Then call `self.send_payment(&payment.sender, payment.token.amount - 1000).await?` (refund minus fee)
   - Then call `self.acknowledge_message(vec![payment.message_id.clone()]).await?` (intentional double-ack — TS does this as safety net, server is idempotent)
   - Return Ok(())

3. `pub async fn list_incoming_payments(&self) -> Result<Vec<IncomingPayment>, MessageBoxError>`:

   - Return collected Vec
