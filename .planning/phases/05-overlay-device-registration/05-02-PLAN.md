<objective>
Wire init-once semantics, upgrade send_message and list_messages for overlay host resolution, add multi-host deduplication, wire host_override through CommsLayer adapter, and add device registration endpoints.

Purpose: Complete the overlay lifecycle — `init()` ensures advertisements exist, `send_message` resolves the recipient's host via overlay, `list_messages` queries all known hosts with dedup, the CommsLayer adapter passes `host_override` through to resolution, and device registration enables FCM push notifications.

CRITICAL: The TS `sendMessage` calls `resolveHostForRecipient(recipient)` for EVERY send when no `overrideHost` is given. Without this, messages go to the wrong host. This plan wires that into the Rust client.
</objective>

   ```rust
   pub(crate) async fn assert_initialized(&self) -> Result<(), MessageBoxError> {
       self.init_once.get_or_try_init(|| async {
           let identity_key = self.get_identity_key().await?;
           let ads = self.query_advertisements(Some(&identity_key), Some(&self.host)).await
               .unwrap_or_default();  // never fail init on overlay query error
           if ads.iter().all(|ad| ad.host.trim() != self.host.trim()) {
               // No matching advertisement — anoint this host
               // CRITICAL TS PARITY: catch anoint errors and continue

               if let Err(e) = self.anoint_host(&self.host).await {
                   // Log but don't fail — client can still function for basic operations
                   eprintln!("Warning: failed to anoint host: {e}");
               }
           }
           Ok::<(), MessageBoxError>(())
       }).await?;
       Ok(())
   }
   ```
